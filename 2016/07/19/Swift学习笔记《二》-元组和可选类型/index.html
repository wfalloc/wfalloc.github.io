Swift学习笔记《二》-元组和可选类型

元组

元组(tuples)把多个值组合成一个复合值。元组内的值可以是任意类型，并不要求是相同类型。

可选类型

可选类型（optionals）用来处理值可能缺失的情况。
常量和变量处于可选类型时，其值可以有值为x 或 没有值。

注意:

C 和 Objective-C 中并没有可选类型这个概念。最接近的是 Objective-C 中的一个特性,一个方法要不返回一个对象要不返回 nil , nil 表示“缺少一个合法的对象”。然而,这只对对象起作用——对于结构体,基本的 C 类型或者枚举类型不起作用。对于这些类型, Objective-C 方法一般会返回一个特殊值(比如 NSNotFound )来暗示值缺失。这种方法假设方法的调用者知道并记得对特殊值进行判断。然而 Swift 的可选类型可以让你暗示任意类型的值缺失,并不需要一个特殊值。

可选类型表示为 Type? 或 Optional(X) 。

nil

给可选类型赋值 nil 表示其没有值。

声明一个可选类型的常量或变量但未初始化，会被自动设置为nil。

注意:

nil 不能用于非可选的常量和变量。如果你的代码中有常量或者变量需要处理值缺失的情况,请把它们声明成对应的可选类型。

Swift 的 nil 和 Objective-C 中的 nil 并不一样。在 Objective-C 中, nil 是一个指向不存在对象的指针。在 Swift 中, nil 不是指针——它是一个确定的值,用来表示值缺失。任何类型的可选状态都可以被设置为 nil ,不只是对象类型。

可选绑定

当一个常量或变量为可选类型时，为了你的程序正常运行，你需要在使用它的时候判断其是否有值。操作一个不存在的可选值可能会导致运行出错。

强制解析(forced unwrapping)

使用 if 来判断一个可选值是否包含值。当你确定可选值包含值，使用 强制解析 ！ 来获取值。

可选绑定（optional binding）

使用可选绑定来判断可选类型是否包含值，如果包含就把值赋给一个临时常量或变量。可选绑定使用在 if while 语句中，用来判断可选类型是否有值，若有值将可选类型的值赋给一个常量或变量。

隐式解析可选类型（implicitly unwrapped optionals）

声明常量或变量时将变量类型后的 ? 替换为 ! ,用来声明一个隐式解析可选类型。

隐式解析类型是可以自动解析的可选类型

隐式解析可选类型适用于可选类型第一次赋值之后会一直有值的情况。
